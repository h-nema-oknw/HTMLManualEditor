# HTMLManualEditor 仕様書

## 1. システム概要

HTMLManualEditorは、HTML形式のマニュアルやドキュメントをWYSIWYG（見たまま編集）で作成・編集できるWebアプリケーションです。
HTML/CSSの知識がなくても、直感的な操作でレイアウトの構築、テキスト編集、画像挿入、テーブル操作などが行えます。

### 1.1 動作環境

| 項目 | 内容 |
|------|------|
| サーバー | IIS（Internet Information Services） |
| ランタイム | 不要（静的ファイルのみ） |
| 対応ブラウザ | Google Chrome（推奨）、Microsoft Edge、Firefox |
| 配置パス | `/HTMLManualEditor/`（IIS wwwroot配下） |

### 1.2 技術スタック

| 項目 | 技術 |
|------|------|
| フレームワーク | React 19 + TypeScript |
| 状態管理 | Zustand |
| UIアイコン | Lucide React |
| CSSフレームワーク | Tailwind CSS（一部）+ カスタムCSS |
| ビルドツール | Create React App (react-scripts) |

## 2. 画面構成

画面は以下の4つのエリアで構成されます。

```
┌────────────────────────────────────────────────┐
│                 ツールバー（上部）                 │
├──────┬───────────────────────┬──────────────────┤
│ 要素  │                      │                  │
│ ツリー │    プレビュー（中央）   │ プロパティ（右側） │
│（左側）│                      │                  │
├──────┴───────────────────────┴──────────────────┤
│               ステータスバー（下部）               │
└────────────────────────────────────────────────┘
```

### 2.1 ツールバー

ファイル操作、書式設定、要素挿入、要素操作、ズームコントロールを含むヘッダー領域。

**ファイル操作:**
- **新規** - 空のドキュメントを作成（変更未保存時は確認ダイアログ）
- **読込** - HTMLファイル（.html / .htm）を読み込み
- **保存** - 現在の内容をHTMLファイルとしてダウンロード
- **PDF** - ブラウザの印刷機能でPDF出力

**元に戻す/やり直し:**
- Ctrl+Z / Ctrl+Y に対応
- 最大50回分の履歴を保持

**書式設定:**
- 太字（B）/ 斜体（I）/ 下線（U）
- テキスト揃え（左/中央/右/両端）
- 箇条書き / 番号付きリスト

**要素挿入（クリック & ドラッグ＆ドロップ対応）:**
- 見出し1（H1）/ 見出し2（H2）/ 見出し3（H3）
- 段落（T）/ 画像 / テーブル
- ブロック / リンク / 水平線
- 2カラム / 3カラム

**要素操作:**
- コピー（複製）/ 上移動 / 下移動 / 削除

**ズーム:**
- 25%〜200% の範囲で10%刻み

### 2.2 要素ツリー（左パネル）

ページ内のDOM構造をツリー形式で表示。
- 要素のタグ名、ID、クラスを表示
- 色分け（div系=紫、見出し=橙、テキスト=緑、テーブル=青緑 等）
- クリックで要素を選択
- 折りたたみ/展開
- 更新ボタンで再描画

### 2.3 プレビュー（中央）

iframe内にHTMLをレンダリングし、WYSIWYG編集を実現。

**表示機能:**
- ホバー時に青い点線枠を表示
- 選択時に青い実線枠を表示
- テキスト編集時に薄ピンクの背景色と赤枠を表示

**操作:**
- クリック → 要素選択
- ダブルクリック → テキスト編集モード
- ドラッグ → 要素の並べ替え（青いラインインジケーター表示）
- ツールバーからのドラッグ＆ドロップ → 要素挿入
- 外側余白クリック → ページ全体（body）選択
- 下部空白クリック → 最終要素選択

### 2.4 プロパティパネル（右パネル）

選択中の要素のスタイルと属性を編集。

**スタイルタブ:**
- レイアウト（幅、高さ、マージン、パディング）
- Flexコンテナ設定（flex要素選択時のみ表示）
- テキスト（フォントサイズ、太さ、色、揃え、行間、字間、フォント）
- 背景・ボーダー（背景色、ボーダー幅/色/形/角丸）

**属性タブ:**
- ID、クラス
- 画像URL / 代替テキスト（img要素選択時）
- リンクURL（a要素選択時）
- テーブル操作ボタン（table関連要素選択時）

**ページ全体選択時:**
- 背景色、文字色、フォント、ページ余白を編集

### 2.5 ステータスバー

- アプリ名・バージョン表示（HTMLManualEditor Ver2）
- 操作ガイドの表示

## 3. 機能仕様

### 3.1 要素の選択

| 操作 | 動作 |
|------|------|
| プレビュー内の要素をクリック | その要素を選択 |
| 要素ツリーの項目をクリック | 対応する要素を選択（スクロール移動） |
| プレビュー外側の余白をクリック | ページ全体（body）を選択 |
| プレビュー下部の空白をクリック | 最終要素を選択 |

### 3.2 テキスト編集

- 要素をダブルクリックでテキスト編集モードに切り替え
- 編集中は薄ピンクの背景色で視覚的に区別
- `contenteditable` 属性を利用
- Delete / Backspace キーで要素削除（テキスト編集中は文字削除）

### 3.3 要素の挿入

**クリック挿入:**
- 入れ物要素（ブロック、カラム等）を選択中 → その中に追加
- 通常要素を選択中 → 選択要素の下に追加
- ページ全体選択中 / 未選択時 → ページ末尾に追加

**ドラッグ＆ドロップ挿入:**
- ツールバーのアイコンをiframe内にドラッグ
- ドロップ位置を青いラインインジケーターで表示
- 要素の上半分 → 前に挿入
- 要素の下半分 → 後に挿入
- 入れ物要素の中央50% → 中に挿入（入れ子配置）
- ページ下部の空白 → 末尾に追加

**カスタムMIMEタイプ:** `application/x-editor-element`

### 3.4 要素の並べ替え（ドラッグ＆ドロップ）

- 選択した要素をマウスドラッグで移動
- 5pxのしきい値を超えた時点でドラッグ開始
- 青いラインインジケーターでドロップ先を表示
- ドラッグ中の要素は半透明表示

**テーブル内の並べ替え:**
- テーブルのセル（td/th）を選択してドラッグ
  - 縦方向のドラッグ → 行の入れ替え
  - 横方向のドラッグ → 列の入れ替え
- ドラッグ方向は自動判定（最初の5px移動方向）
- ピンク色のインジケーターで入れ替え先を表示
- テーブル全体の移動は左上のハンドルをドラッグ

### 3.5 入れ子構造（ネスト）

- ブロック、2カラム、3カラムなどの入れ物要素の中に子要素を配置可能
- **制限:** 2階層まで（親子関係）。孫要素の配置は不可
- 深さの計算は `editor-table-wrapper`（エディタ内部用）をスキップ
- 対応する入れ物要素: `div`, `section`, `article`, `aside`, `nav`, `header`, `footer`, `main`, `td`, `th`, `li`, `blockquote`, `details`, `figure`, `figcaption`

### 3.6 テーブル操作

| 操作 | 方法 |
|------|------|
| テーブル全体選択 | 左上のハンドル（⊞）をクリック |
| テーブル移動 | 左上のハンドルをドラッグ |
| 行追加 | テーブル下端の「＋」ボタン or プロパティパネル |
| 列追加 | テーブル右端の「＋」ボタン or プロパティパネル |
| 行削除 | プロパティパネルの「行削除」ボタン |
| 列削除 | プロパティパネルの「列削除」ボタン |
| セル編集 | セルをダブルクリック |
| 行の入れ替え | セルを選択して縦方向にドラッグ |
| 列の入れ替え | セルを選択して横方向にドラッグ |

### 3.7 画像の挿入

**クリック挿入:**
1. ツールバーの画像アイコンをクリック
2. ファイル選択ダイアログが表示
3. 画像を選択 → Base64形式でiframe内に挿入

**ドラッグ挿入:**
1. ツールバーの画像アイコンをドラッグ＆ドロップ
2. プレースホルダー（「クリックして画像を選択」）が配置される
3. プレースホルダーをクリック → ファイル選択ダイアログ
4. 画像選択 → プレースホルダーが実画像に置換

**OSファイルドロップ:**
- ブラウザのファイルをiframe内にドロップ → 画像として挿入

**画像サイズ:**
- 親要素の幅に合わせて `max-width: 100%` で自動縮小

### 3.8 元に戻す / やり直し

- 最大50回分の履歴を保持（リングバッファ方式）
- Ctrl+Z / Ctrl+Y キーボードショートカット対応
- ツールバーのボタンからも操作可能
- 新しい操作を行うとやり直しスタックはクリア
- 500msのデバウンスで編集をグループ化

### 3.9 ズーム

- 25% 〜 200% の範囲
- 10%刻みで増減
- CSSの `transform: scale()` で実現
- ズーム時にiframeの幅を逆スケールで補正

## 4. データフロー

```
ユーザー操作
    ↓
iframe内のイベントハンドラ（iframeManager）
    ↓
DOM変更を検知（MutationObserver）
    ↓
デバウンス（500ms）
    ↓
serializeIframeToHtml() でクリーンなHTMLを抽出
    ↓
Zustand Store（htmlContent）を更新
    ↓
要素ツリー再描画 / 履歴に記録
```

### 4.1 HTMLシリアライズ

iframe内のDOMをクリーンなHTMLに変換する際、以下を除去:
- エディタ注入要素（`data-editor-injected`）
- テーブルラッパー（`.editor-table-wrapper` → アンラップ）
- エディタ属性（`data-editor-selected`, `data-editor-hover`, `contenteditable`）
- エディタ用CSSクラス（`editor-drop-inside`, `editor-dragging`）
- エディタ用インラインスタイル（outline, cursor等）

## 5. ファイル入出力

### 5.1 入力（読込）

- ファイル選択ダイアログから `.html` / `.htm` ファイルを選択
- `FileReader` で読み込み、iframeに反映
- 履歴はクリアされ、初期状態として記録

### 5.2 出力（保存）

- iframe内のDOMをシリアライズ
- エディタ固有の要素・属性を全て除去
- `<!DOCTYPE html>` を先頭に付与
- Blob経由でHTMLファイルとしてダウンロード
- ファイル名は現在のドキュメント名 + `.html`

### 5.3 PDF出力

- ブラウザの `window.print()` を利用
- 印刷ダイアログから「PDFとして保存」を選択

## 6. 初期表示

アプリケーション起動時は、操作マニュアル（HTMLManualEditorの使い方を説明するHTMLドキュメント）がデフォルトで表示されます。
ユーザーはこのマニュアルを直接編集して操作を体験できます。

「新規」ボタンで空のドキュメントに切り替え可能です。

## 7. デプロイ手順

### 7.1 ビルド

```bash
cd editor-app
BUILD_PATH=./output npx react-scripts build
```

### 7.2 配置

ビルド成果物（`editor-app/output/` 内の全ファイル）を IIS の以下のパスにコピー:

```
C:\inetpub\wwwroot\HTMLManualEditor\
```

### 7.3 アクセス

```
http://<サーバー名>/HTMLManualEditor/
```

### 7.4 配置パスの変更

`package.json` の `homepage` フィールドを変更して再ビルド:

```json
"homepage": "/新しいパス/"
```

## 8. 制約事項・注意点

- 画像はBase64エンコードでHTMLに埋め込まれるため、多数の大容量画像を挿入するとファイルサイズが大きくなります
- iframeの `sandbox` 属性により、iframe内からのファイル選択ダイアログ表示は制限されるため、親ドキュメント経由で実行しています
- 入れ子構造は2階層までに制限されています
- テーブルのセル単体の移動はサポートしていません（行・列単位での入れ替えのみ）
- ブラウザのローカルストレージへの自動保存機能はありません。定期的に「保存」ボタンでファイルをダウンロードしてください
