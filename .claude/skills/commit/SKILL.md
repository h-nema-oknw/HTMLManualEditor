# 変更コミット

変更内容を分析し、適切な日本語コミットメッセージを生成してコミットする。

## 手順

### 1. 変更内容の確認

```bash
git status
git diff --stat
git diff
```

### 2. 変更の分析

変更内容を読み取り、以下を判断する：
- 変更の性質（新機能追加 / 機能改善 / バグ修正 / リファクタリング / ドキュメント更新）
- 影響範囲（どのコンポーネント・モジュールに変更があるか）

### 3. コミット対象の選定

以下のファイルはコミット対象から**除外**する：
- `editor-app/output/` （ビルド成果物）
- `editor-app/build*/` （レガシービルド）
- `node_modules/`
- `.env` やクレデンシャル系ファイル

関連するファイルを個別に `git add` でステージングする。

### 4. コミットメッセージの生成

日本語で簡潔なコミットメッセージを作成する。形式：

```
#<概要>

<詳細（必要に応じて）>
```

例：
- `#フローティングシェイプ機能を追加`
- `#テーブル操作のバグを修正`
- `#PropertiesPanelのスタイル編集を改善`

### 5. コミット実行

ステージング済みファイルをコミットする。コミット前にユーザーにメッセージを確認してもらう。

## 注意事項

- コミットメッセージは日本語で記述する
- 大きな変更は機能単位で分割コミットを提案する
- コミット前に必ずユーザーの確認を取る
